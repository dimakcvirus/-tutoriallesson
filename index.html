010 Учимся работать с системой контроля версий Git и с сервисом GitHub

git и github это две разные программы.

https://git-scm.com/ от сюда устанавливаем программу.

Что бы узнать польностю git есть отдельный курс githowto.com.

Первое что нужно сделать это сказать гиту что бы он следил за папкай в которой мы работаем командой git init

Второе есть глобальные настройки и локальные это надо почитать что это такое.

Команда для локальных настроек git config –local user.name“Dmitry“
            git config —local user.email dimakc@mail.ru

после первой установки эти настройки нужно сделать глобально 

git config –global user.name“Dmitry“
            git config —global user.email dimakc@mail.ru

что бы проверить статус пишем git status

мы увидим что есть файлы которые не добавлены в репозиторий для сохранения что бы их добавить нужно написать. Git add -A „ где A — это добавления сразу всех файлов“. Что бы добавить конкретный файл нужно написать git add и имя файла если мы сейчас еще раз напишеи гит статус то увидим что файлы добавлены в репозиторий.

Теперь что бы начать бэкапить эти все файлы нужно написать git commit -a -m « сюда названия коммита например это мой первый коммит » опять же -a это мы берем все файлы, -m обязательно ковычки двойные и пишем имя коммита. Что бы увидеть какие коммиты и сколько их мы сделали есть команда git log.

Есть возможнотсь соеденить git и github для этого пишим команду git remote add origin https: и ссылку на репозиторий из нашего гитаба нажимаем интер содеденение произошло теперь чтобы сохранить наш локальный проекат на гитхабе пишем команду git push -u origin master
